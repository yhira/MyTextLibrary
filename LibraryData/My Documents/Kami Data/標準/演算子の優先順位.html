演算子の優先順位

　数学の問題です。次の答えは何になるでしょうか。

    8 - 2 + 10 ÷ 5

　正解は、8です。まさか間違えた方はいないでしょう。数学では、わり算・かけ算を先に計算し、次に足し算・引き算を計算するというルールがあります。また、足し算・引き算同士の場合、左側から順番に計算します。
　上の例では、まず10÷5を計算し、次に8-2、最後に8-2の答えと、10÷5の答えを足します。
　Pascalでも同じで、どの演算子から優先的に計算するのか決まっています。この優先順位は次のとおりです。
　表の上にあるものから順番に計算し、同じ優先順位の演算子は、式の左側から計算されます。

優先順位 演算子
１ not
２ *　/　div　mod　and
３ +　-　or　xor
４ =　<>　<　>　<=　>=　in

　例えば次の式の結果は、95になります。

  Ans := 100 - 50 div 10;

　計算する順序を変えたいときは、( )カッコで囲みます。例えば上記の式で減算を先に計算したいときは、次のようにします。

  Ans := (100 - 50) div 10;

　数学と同様にカッコで囲まれた部分が先に計算されます。上記の式の結果は、5になります。複雑な式を計算する場合、計算の順番が分かりづらいときは、わざとカッコを付けて分かりやすくする手もあります。

  Ans := 100 div 50 - 115 mod 10;      { このような式は }
  Ans := (100 div 50) - (115 mod 10);  { わざとカッコを付けると分かりやすくなることもある }

　優先順位で特に注意したいのは、if文の条件の部分です。次のif文はエラーになります。

  if A = 5 and B = 0 then

　理由は、andの優先順位が高いため、まず5 and Bが計算されてしまうためです。このようなことを避けるため、if文に論理演算子を用いるときは、必ずカッコで囲みます。

  if (A = 5) and (B = 0) then
